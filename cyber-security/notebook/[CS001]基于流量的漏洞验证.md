# 基于流量的漏洞检测的三个阶段

基于流量对漏洞进行检测可以分为三个阶段：

1. 数据清洗：清洗出需要的URL，并通过模型过滤那些不需要检测的URL；
2. 漏洞扫描：对这些重点URL进行扫描尝试发现漏洞；
3. 结果运营：通过运营进一步去除误报的URL、确认漏洞的危害、是否有进一步利用的可能、以及其他相关接口是否还存在相同的漏洞，用来反哺修正扫描器；[^1]
